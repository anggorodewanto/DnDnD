## Acceptance criteria
- Every `RegisterRoutes(router)` mount listed above is moved below the `dmAuthMw` construction in `cmd/dndnd/main.go` and wrapped in a `router.Group` that applies `dmAuthMw`.
- An authenticated non-DM Discord user hitting any of the mutation endpoints (e.g. `POST /api/campaigns/{id}/pause`, `POST /api/encounters/{id}/advance-turn`, `POST /api/maps/import`) receives 403.
- A regression test (table-driven) walks every mounted DM-mutation route and asserts 403 for non-DM and 2xx for DM.
- Tests pass; `make cover-check` green.
