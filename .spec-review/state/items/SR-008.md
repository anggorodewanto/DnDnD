## Finding (verbatim from review)
Fog of war is dead code in production. `cmd/dndnd/discord_adapters.go:417-514` calls `ComputeVisibilityWithLights` with empty `VisionSources` / `LightSources` / `MagicalDarknessTiles`; `DMSeesAll` never set. Full shadowcasting algorithm + tests exist but the `if len(md.VisionSources) > 0` guard is always false. Maps render fully uncovered. Largest single user-facing gap.

## Acceptance criteria
- `RegenerateMap` (in `cmd/dndnd/discord_adapters.go`) populates `MapData.VisionSources` from active combatants/PCs, `LightSources` from lit torches / spells / scene lighting, `MagicalDarknessTiles` from active zones, and `DMSeesAll = true` when the renderer is producing the DM view.
- After populating, `if len(md.VisionSources) > 0` is true on a real encounter and `ComputeVisibilityWithLights` is invoked.
- Integration test: a renderered PC map for a scene with one PC + one torch hides tiles outside both vision radius and light radius; the DM map shows everything.
- Tests pass; `make cover-check` green.
