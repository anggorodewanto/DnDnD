## Finding (verbatim from review)
`/action help` advantage is silently broken. `standard_actions.go:250-257` applies `help_advantage` condition, but no read site in `advantage.go`/`attack.go` consumes it. Help grants zero benefit.

## Acceptance criteria
- `advantage.go` (or `attack.go`) reads the `help_advantage` condition on the helped creature and grants advantage on its next attack vs the spec'd target, then clears the condition after the attack.
- Test: ally `/action help` on a creature → that creature's next attack vs the named target rolls with advantage; the second attack does not.
- Tests pass; `make cover-check` green.

## Worker plan

### Design
- Extend `CombatCondition` (`initiative.go`) with `TargetCombatantID string` (json `target_combatant_id,omitempty`). Mirror of existing `SourceCombatantID` / `SourceSpell` precedent — `help_advantage` is the first user but the field is generic.
- `standard_actions.go:Help` populates `TargetCombatantID = cmd.Target.ID.String()` when applying `help_advantage`.
- `advantage.go`: extend `AdvantageInput` with `TargetCombatantID string`. In the attacker-conditions loop, treat `help_advantage` as an advantage source iff `c.TargetCombatantID == input.TargetCombatantID` (target-scoped). Reason: `"help advantage"`.
- `attack.go`: in `ResolveAttack`, plumb `input.TargetCombatantID` into `AdvantageInput`. `buildAttackInput` already takes attacker+target combatants, so populate `TargetCombatantID = target.ID.String()` there.
- Consumption (single-shot): in `Service.Attack` (and mirror in `attackImprovised`), AFTER `resolveAndPersistAttack` succeeds, check whether the attacker has a `help_advantage` condition scoped to this target — if yes, `RemoveCondition` from the attacker's row. Use a small helper `consumeHelpAdvantage(ctx, attacker, target)` to keep both call sites tidy. Early-return style.
- Note: target-scoping means a Help against goblin-A does NOT grant advantage on an attack vs goblin-B (test 3). Single-shot consumption means attack #2 vs the same target gets no advantage (test 2).
