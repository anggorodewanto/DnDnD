## Finding (verbatim from SR-008 reviewer)
Two non-blocking gaps surfaced while approving SR-008:

1. `RegenerateMapForDM` has zero production callers ‚Äî DM-view PNG path still needs `/dashboard/encounters/{id}/map.png?view=dm` (or equivalent), so the DM still sees a fogged map.
2. Empty `LightSources` plus `defaultBaseSightTiles=12` (60ft) means non-darkvision PCs see 60ft everywhere ‚Äî darkness encounters do not feel dark. Needs a production data source for lit torches / Light / Daylight / Continual Flame / scene lighting.

## Acceptance criteria
- A DM-facing render endpoint (route + handler) calls `RegenerateMapForDM` so the DM PNG shows the unfogged map.
- `LightSources` is populated from a real data source: lit inventory items (torch / lantern with `is_lit` or equivalent state), active Light/Daylight/Continual Flame spell instances, and/or scene lighting from the encounter row.
- Integration test: non-darkvision PC in a scene with one lit torch ‚Äî tiles outside torch radius are dark; without torch, tiles outside the small base-sight radius are dark.
- DM-view test: same scene, DM endpoint renders everything visible.
- Tests pass; `make cover-check` green.

## Worker plan
- Add `IsLit` field to `character.InventoryItem` so torches/lanterns can be marked as lit.
- Add `buildLightSources` function in `discord_adapters.go` that reads lit inventory items from PC combatants' characters and active light-emitting zones (Light cantrip, Daylight, Continual Flame) to produce `[]renderer.LightSource`.
- Add Light/Daylight/Continual Flame zone definitions in `zone_definitions.go` with `zone_type: "light"`.
- Wire `buildLightSources` into `renderInternal` so `md.LightSources` is populated before the FoW guard.
- Add DM map PNG handler in `dashboard_apis.go` and mount at `GET /api/combat/{encounterID}/map.png` behind dmAuthMw; handler calls `RegenerateMapForDM`.
- Write failing tests: (a) `buildLightSources` returns sources from lit torches, (b) integration test with non-darkvision PC + torch showing dark tiles outside radius, (c) DM-view endpoint returns unfogged PNG.
- Run `go test ./...` and `make cover-check`.
