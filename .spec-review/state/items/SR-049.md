## Finding (verbatim from review)
Summons always share caster's turn — no independent-initiative mode.

## Acceptance criteria
- Summons can be configured (per spell or DM setting, per spec) to take their own initiative slot rather than always sharing the caster's turn.
- Test: a summon set to independent initiative is inserted into the turn order at its own roll.
- Test: a summon set to share-caster-turn behaves as today.
- Tests pass; `make cover-check` green.

## Worker plan
- Add `SharesCasterTurn(c)` helper: true when summoned AND `InitiativeOrder == 0` (never given own slot)
- Replace `IsSummonedCreature(c)` with `SharesCasterTurn(c)` in `AdvanceTurn` candidate filtering and `findFirstActiveCombatant`
- Update `RollInitiative` filter to use `SharesCasterTurn` so pre-combat independent summons get rolled
- Add `InsertSummonIntoInitiative(ctx, encounterID, combatantID, roller)` to roll and insert a summon mid-combat
- Add `IndependentInitiative bool` to `SummonCreatureInput`; when true, set a non-zero `InitiativeOrder` placeholder (actual roll deferred to `InsertSummonIntoInitiative`)
- Write unit tests: independent-initiative summon gets own turn; caster-turn summon behaves as before
- Verify `make cover-check` passes
