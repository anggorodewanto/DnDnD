## Finding (verbatim from review)
Server's `generateDefaultTiledJSON` produces 2 layers vs editor's 5. Maps created via raw `POST /api/maps` are missing lighting/elevation/spawn_zones. Becomes a bug once item above is fixed.

## Acceptance criteria
- `generateDefaultTiledJSON` emits all 5 layers: `terrain`, `walls`, `lighting`, `elevation`, `spawn_zones`.
- Layer schema matches what the editor produces and what `renderer/parse.go` (post-SR-029) reads.
- Test: `POST /api/maps` produces JSON the renderer parses without errors and yields the same layer count as the editor.
- Tests pass; `make cover-check` green.
