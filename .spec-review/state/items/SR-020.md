## Finding (verbatim from review)
Auto-skip Dodge never expires. `timer_resolution.go:194-199` applies Dodge with `ExpiresOn: "start_of_next_turn"` and no `SourceCombatantID`; `isExpired` only recognizes `start_of_turn`/`end_of_turn` AND requires source.

## Acceptance criteria
- Auto-skip Dodge is applied with an `ExpiresOn` value `isExpired` actually recognises, and with a non-nil `SourceCombatantID` (the creature dodging).
- Test: auto-skip Dodge applied on a creature's turn expires at the start of its next turn.
- Tests pass; `make cover-check` green.

## Worker plan

Root cause:
- `timer_resolution.go:194-199` (auto-skip Dodge in `AutoResolveTurn`) builds a `CombatCondition` with `ExpiresOn: "start_of_next_turn"` and no `SourceCombatantID`. `isExpired` (in `condition.go:72-91`) only recognises `start_of_turn` / `end_of_turn` and requires `SourceCombatantID == triggerCombatantID`. Result: auto-skip Dodge never matches and never clears, so it stays on the combatant forever.
- Hand-rolled `/action dodge` (`standard_actions.go:153-202`) already uses `ExpiresOn: "start_of_turn"` + `SourceCombatantID: cmd.Combatant.ID.String()` — same shape as the rest of the codebase (Help, Rage, attack-grants).

Fix shape (minimal):
1. In `timer_resolution.go` `AutoResolveTurn`, swap the literal `"start_of_next_turn"` for `"start_of_turn"` and set `SourceCombatantID: combatant.ID.String()` so the condition matches the same expiry path as the `/action dodge` happy path.
2. Add red-then-green tests in `timer_resolution_test.go`:
   - `TestAutoResolveTurn_DodgeExpiresAtStartOfNextTurn`: auto-skip applied on round R, then run `CheckExpiredConditions` with `triggerCombatantID = combatant.ID` and `currentRound = R+1` → expired list contains "dodge", remaining is empty.
   - `TestAutoResolveTurn_DodgePersistsThroughEnemyTurn`: same state, but `triggerCombatantID = "someone-else"` (enemy) → not expired; remaining still has "dodge". This proves the source-scoping is correct so the condition survives enemies' turns and only clears when the dodging creature starts their next turn.

Red/green sequence:
- Red: write both tests asserting the new shape (`ExpiresOn = "start_of_turn"`, non-empty `SourceCombatantID == combatant.ID`, expiry behaviour). They fail against the current code because `ExpiresOn` is `"start_of_next_turn"` and source is empty.
- Green: apply the two-line edit in `timer_resolution.go`.
- Verify: `go test ./internal/combat/...`, then full `go test ./...`, then `make cover-check`.
