## Finding (verbatim from review)
`Bot.HandleGuildCreate` / `HandleGuildMemberAdd` never wired; no `Identify.Intents`. `cmd/dndnd/main.go:1197-1211` only adds `InteractionCreate`. Welcome DMs (spec §183–200) and dynamic guild-join command registration (spec §179) silently inert.

## Acceptance criteria
- `session.AddHandler(bot.HandleGuildCreate)` and `session.AddHandler(bot.HandleGuildMemberAdd)` are registered alongside the existing `InteractionCreate` handler.
- `session.Identify.Intents` includes `IntentsGuildMembers` (and any others required for `GuildCreate` / `GuildMemberAdd`).
- A test asserts the registered handlers list contains both new handlers and that `Identify.Intents` is non-zero with `GuildMembers` flag set.
- Tests pass; `make cover-check` green.

## Worker plan

Wiring at `cmd/dndnd/main.go:1297-1311` only adds `InteractionCreate`; `Identify.Intents` is never set so the discordgo default `IntentsAllWithoutPrivileged` excludes `GuildMembers`.
