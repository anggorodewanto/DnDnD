## Finding (verbatim from review)
Turned condition does not auto-clear on damage (spec rule).

## Acceptance criteria
- Applying damage to a creature with the Turned condition clears the condition per spec rule.
- Test: undead is Turned → it takes damage → Turned condition is removed.
- Test: damage from the turning creature (or others) both trigger the auto-clear per spec.
- Tests pass; `make cover-check` green.

## Worker plan
1. Extend the existing post-`applyDamageHP` hook block in `Service.ApplyDamage`
   (the same seam used by `markRageTookDamage` /
   `maybeEndRageOnUnconscious`). Add a single helper
   `maybeClearTurnedOnDamage(ctx, target, adjusted)` that, when `adjusted > 0`
   and the target still has the `turned` condition, calls
   `RemoveConditionFromCombatant`. This keeps the change inside the existing
   post-damage-effects hook block — no new pipeline stage, no caller-side
   sprinkling.
2. Gate strictly on `adjusted > 0` so healing (which is not routed through
   `ApplyDamage` anyway) and immunity / full-temp-HP absorption (`adjusted ==
   0`) do not trigger clearing.
3. Source-agnostic: the helper inspects the target's conditions, not the
   damage source — both turner damage and third-party damage clear the
   condition, matching the spec rule.
4. Best-effort: lookup / persistence errors are swallowed (consistent with
   `markRageTookDamage` / `maybeEndRageOnUnconscious`) so the damage write
   never blocks on cleanup.
5. Red/green TDD via `damage_test.go`:
   - `TestApplyDamage_ClearsTurnedConditionOnDamage` — undead with `turned`
     takes any positive damage → `turned` removed via
     `UpdateCombatantConditions`.
   - `TestApplyDamage_ClearsTurnedFromTurnerOrThirdParty` — neither path
     consults the source; same removal happens regardless of who deals damage.
   - `TestApplyDamage_DoesNotClearTurnedOnZeroAdjustedDamage` — immune target
     (adjusted = 0) keeps `turned`.
   - `TestApplyDamage_DoesNotClearTurnedWhenFullyAbsorbedByTempHP` — temp-HP
     fully absorbs damage (adjusted = 0) keeps `turned`. Healing is not
     routed through ApplyDamage (no separate negative-damage path); the
     zero-adjusted cases cover the "no HP loss → no clear" invariant.
6. `make cover-check` to verify coverage stays green.
