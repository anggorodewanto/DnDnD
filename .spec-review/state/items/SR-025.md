## Finding (verbatim from review)
Metamagic interactive prompts are dead code. `MetamagicPromptPoster.PromptEmpowered/Careful/Heightened` are built and unit-tested but never invoked by `cast_handler.go`. `CastCommand.TwinTargetID` is never read from Discord.

## Acceptance criteria
- `cast_handler.go` invokes `MetamagicPromptPoster.PromptEmpowered`/`PromptCareful`/`PromptHeightened` whenever the caster selects the corresponding metamagic option.
- `CastCommand.TwinTargetID` is parsed from the Discord interaction options and threaded into the cast pipeline.
- Test: sorcerer casts with Empowered → prompt is posted and consumed; cast proceeds with the empowered reroll.
- Test: Twinned spell with a second target uses `TwinTargetID`.
- Tests pass; `make cover-check` green.

## Worker plan

### Goals
1. Wire `MetamagicPromptPoster.PromptEmpowered` / `PromptCareful` / `PromptHeightened`
   into `cast_handler.Handle()` (single-target + AoE) so a Sorcerer selecting
   the corresponding metamagic flag actually sees the interactive prompt.
2. Parse a `twin-target` Discord option in `cast_handler.go` and thread it
   into `CastCommand.TwinTargetID` (currently always `uuid.Nil`).
3. Execute the Empowered reroll server-side. SR-013 deferred this — the spell
   announces "may reroll up to N dice" but no reroll happens. Add a helper +
   plumb it through `ResolveAoESaves` so AoE-cast damage actually re-rolls the
   N lowest dice once per Empowered cast.
4. Pin the production wire-up via a wiring test mirroring SR-022 / SR-024
   (the MetamagicPromptPoster must be reachable from `buildDiscordHandlers`).

### Notes / minimality
- Keep one prompt per option (PromptEmpowered's "up to N" multi-pick is the existing per-die loop; for SR-025 we collapse to a single click — the server-side reroll auto-applies to the N lowest dice regardless).
- Careful prompt re-posts up to `CHA mod` times (canonical from existing impl), but for the minimal first cut we do **one** prompt and let the user pick one ally; subsequent allies can be added in a follow-up if needed.
- The prompt's `Forfeited` path proceeds with the cast unchanged (Empowered still auto-applies to the N lowest because the server-side reroll is independent of the prompt click; Careful with no click → no allies protected; Heightened with no click → first-affected default).
