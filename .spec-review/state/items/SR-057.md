## Finding (verbatim from review)
`errorlog` only fires on panic, not normal error returns (Batch 16); 24h dashboard badge will under-report.

## Acceptance criteria
- `errorlog` (or equivalent) captures normal error returns from handler code paths, not just panics.
- 24h dashboard badge reflects all errors, not just panics.
- Test: a handler returning an error increments the errorlog count visible to the dashboard.
- Tests pass; `make cover-check` green.

## Worker plan
- Add `ErrorReportingHandler` interface in router.go: `HandleCommand(*discordgo.Interaction) error`
- In router's `Handle` dispatch, type-assert handler to `ErrorReportingHandler`; if yes, call `HandleCommand` and record non-nil error to errorlog + send friendly ephemeral
- Fall back to existing `Handle(interaction)` for handlers that don't implement the new interface (backward-compatible)
- Write failing test: handler implementing `ErrorReportingHandler` returns error → errorlog count increments
- Make test pass with minimal router change
- Run `go test ./...` and `make cover-check`
