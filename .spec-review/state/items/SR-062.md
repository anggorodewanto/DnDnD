## Finding (verbatim from review)
Session cookie MaxAge doesn't slide with DB TTL.

## Acceptance criteria
- On each authenticated request, the session cookie `MaxAge` and the DB session TTL extend together (sliding window per spec).
- Test: making a request near the TTL boundary refreshes both the cookie and the DB row; a quiet session expires at the spec'd horizon.
- Tests pass; `make cover-check` green.

## Worker plan
- Add `secure bool` parameter to `SessionMiddleware` so it can build a cookie.
- After `SlideTTL` succeeds, call `http.SetCookie` with `CookieName`, same value, `MaxAge = SessionTTL`, matching attributes (HttpOnly, Secure, SameSite=Lax, Path=/).
- Update all callers of `SessionMiddleware` (main.go, tests) to pass the new param.
- Add a test `TestSessionMiddleware_SlidesCookieMaxAge` that asserts the response contains a `Set-Cookie` header with refreshed MaxAge.
- Run `go test ./...` and `make cover-check`.

## Files touched

- `internal/auth/middleware_test.go` — Added `TestSessionMiddleware_SlidesCookieMaxAge` and `_Secure` tests; updated all existing calls to pass `secure` param.
- `cmd/dndnd/main.go` — Pass `secure` flag to `SessionMiddleware`.
