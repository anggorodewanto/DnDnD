## Finding (verbatim from review)
`/equip` bypasses `combat.Equip` entirely. `internal/discord/equip_handler.go:91` â†’ `inventory.Equip`, which only flips a JSONB `Equipped` flag. No AC recalculation, no 2H/shield validation, no in-combat armor block, no writes to `equipped_main_hand/off_hand/armor` columns that grapple/somatic/stealth/attack all read. The fully-tested `combat.Equip` service is dead.

## Acceptance criteria
- `internal/discord/equip_handler.go` routes through `combat.Equip` (or equivalent service that performs AC recalc, 2H/shield validation, in-combat armor block, column writes).
- After `/equip`, `equipped_main_hand`, `equipped_off_hand`, and `equipped_armor` columns are written so downstream grapple/somatic/stealth/attack code sees the change.
- Negative test: equipping armor while in combat is rejected with the spec'd error.
- Two-handed and shield validation rejects illegal combinations.
- Tests pass; `make cover-check` green.

## Worker plan
- **Re-architect `internal/discord/equip_handler.go`** to depend on a new
  `EquipCombatService` interface that exposes `Equip(ctx, combat.EquipCommand)
  (combat.EquipResult, error)`, plus encounter/turn lookups
  (`ActiveEncounterForUser`, `GetEncounter`, `GetTurn`). `*combat.Service`
  satisfies the operation; the resolver+queries satisfy the lookups. Keep the
  existing inventory JSON `Equipped`/`EquipSlot` write (in-handler) so magic
  item bonuses, portal display, and DDB import continue to read the flag â€”
  `combat.Equip` only writes columns. Also support `/equip none` by translating
  it to the combat path (already covered there).
- **Detect in-combat state**: resolve active encounter for invoker; if found
  AND `encounter.CurrentTurnID.Valid`, load the turn and pass it to
  `combat.Equip`. If no active encounter, call with nil turn (out-of-combat).
  This single code path lets `combat.Equip` apply free-interact / shield-action
  / armor-block validation cleanly.
- **Wire in `cmd/dndnd/discord_handlers.go:434`**: replace the
  `NewEquipHandler(... deps.queries)` call with a new constructor signature
  that takes `combatService` and a `combatActionLookupAdapter`-style
  encounter+turn provider. Reuse the existing `combatActionLookupAdapter`
  (already provides `ActiveEncounterForUser`, `GetEncounter`, `GetTurn`).
  Guard on `deps.combatService == nil` for test deploys â€” fall back to the old
  out-of-combat-only mode so unit tests without combat svc still register the
  handler.
- **Tests** in `internal/discord/equip_handler_test.go` (redâ†’green):

  (b) `/equip greatsword` with shield in off-hand is rejected with "off-hand

  (d) Successful armor equip records `NewAC` from the combat result in the
     ephemeral reply (proxy for "AC was recalculated"); also assert the mock
     combat svc was invoked instead of the old inventory-only path.
- **Coverage**: `combat.Equip` is already covered by `equip_test.go` (45+
  tests). New work focuses on handler-side wiring + interface satisfaction.
- **Out of scope per task brief**: do NOT add `OnCharacterUpdated` from
