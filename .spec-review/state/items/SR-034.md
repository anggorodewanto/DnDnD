## Finding (verbatim from review)
Ability-score generation methods missing across portal + dashboard. Spec wants point-buy / standard array / roll gated by DM setting; portal only does point-buy; dashboard only manual entry.

## Acceptance criteria
- Portal supports point-buy, standard array, and roll (4d6 drop lowest) modes; selection is gated by the campaign's DM-controlled setting.
- Dashboard char creation supports the same three modes per the DM setting.
- Test: each mode produces ability scores consistent with the spec rules; selecting a mode disallowed by the DM setting is rejected.
- Tests pass; `make cover-check` green.

## Worker plan
- Add shared ability-generation validation for point-buy, standard array, and 4d6-drop-lowest roll submissions.
- Read allowed ability generation methods from campaign settings, defaulting to the existing point-buy/manual-compatible behavior.
- Add red tests for portal and dashboard service validation, including disallowed method rejection.
- Update portal and dashboard submissions to carry ability method metadata.
- Add minimal UI/API support so each creator can select only campaign-allowed modes.
- Retry: require roll submissions to include all six 4d6-drop-lowest dice records and derive omitted dashboard API method as point-buy for campaign gate checks.
- Run Go tests, relevant frontend tests, and `make cover-check`.
