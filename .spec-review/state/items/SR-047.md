## Finding (verbatim from review)
Drag-while-moving / Release-and-Move interactive prompt is decorative only (separate `/bonus release-drag`).

## Acceptance criteria
- The interactive prompt for drag-while-moving / release-and-move actually drives the relevant state change (drop the dragged target, end the grapple, continue movement) rather than relying on a separate `/bonus release-drag` command.
- Test: clicking "Release" on the prompt ends the drag and moves the character to the chosen tile in one flow.
- Tests pass; `make cover-check` green.

## Worker plan
- Extend `MoveDragLookup` interface to include `ReleaseDrag` (or add a new `MoveDragReleaser` interface on the handler).
- When `/move` detects drag targets, show a two-button prompt (Drag / Release & Move) with custom ID prefix `drag_choice:` encoding turn/combatant/dest/cost data.
- Add router dispatch for `drag_choice:drag:...` → existing doubled-cost confirm flow.
- Add router dispatch for `drag_choice:release:...` → call `ReleaseDrag`, then show normal-cost confirm.
- Add failing test: clicking "Release & Move" ends grapple and shows normal-cost confirmation.
- Add failing test: clicking "Drag" shows doubled-cost confirmation.
- Run `go test ./...` and `make cover-check`.
