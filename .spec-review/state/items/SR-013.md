## Finding (verbatim from review)
AoE cylinder shape unsupported + AoE pipeline ignores Metamagic. `internal/combat/aoe.go:226-238` has no `cylinder` case (silently breaks Moonbeam, Flame Strike, Ice Storm, Sleet Storm, Call Lightning, Reverse Gravity). `AoECastCommand` has no `Metamagic` field at all — Careful/Heightened/Twinned never reach AoE casts.

## Acceptance criteria
- `aoe.go` switch covers the `cylinder` shape (radius + height) with correct tile selection.
- `AoECastCommand` carries a `Metamagic` field and the AoE pipeline applies Careful (exclude allies), Heightened (impose disadvantage on save), Twinned (where legal), and other applicable metamagic flags.
- Tests: Moonbeam selects a cylinder; Flame Strike likewise; Careful + Fireball spares an ally tile from saving-throw effects.
- Tests pass; `make cover-check` green.

## Worker plan

### Geometry: `cylinder`
- For a top-down 2D grid the column footprint of a cylinder of radius R and height H is **the same disc as a sphere of radius R**. Height_ft is decorative for tile selection — it matters only for vertical clearance / fog / line-of-effect (out of scope for SR-013, deferred to SR-008/SR-014).
- Seeded spell data uses `{"shape":"cylinder","radius_ft":R,"height_ft":H}`. `AreaOfEffect` already exposes `RadiusFt`; `HeightFt` does not yet exist in the struct and is intentionally **not added** (no current code needs vertical reasoning; YAGNI).
- Implementation: add a `case "cylinder"` to `GetAffectedTiles` that delegates to `SphereAffectedTiles(targetCol, targetRow, aoe.RadiusFt)`. No new helper needed (a cylinder's grid footprint == sphere disc). Origin for cover is the target tile (same as sphere), so the existing cover branch in `CastAoE` continues to work unchanged for cylinder.

### `AoECastCommand.Metamagic` shape
- Mirror `CastCommand`'s shape:
  - `Metamagic []string` — list of normalized option names.
  - `CarefulTargetIDs []uuid.UUID` — explicit ally combatant IDs to auto-succeed (set by the eventual SR-025 prompt UX; for now the field is accepted and honored if non-empty).
  - No `TwinTargetID` — Twinned is rejected for AoE (spec + `validateTwinnedSpell` already rejects AoE shapes).
- AoE-eligible options per spec/`metamagic.go`: **Careful** (auto-success allies), **Heightened** (first save disadvantage), **Empowered** (re-roll lowest damage dice), **Distant** (range scaling — already works for AoE since cover/range read from `ValidateSpellRange`), **Extended**, **Subtle**, **Quickened** (bonus-action timing). **Twinned** is rejected by `validateTwinnedSpell` since AoE spells have `area_of_effect`.

### Pipeline plumbing
1. In `CastAoE`, after slot/silence validation but **before** slot deduction:
   - Parse `feature_uses["sorcery-points"]` (via `ParseFeatureUses`).
   - Run `ValidateMetamagic` (one-per-spell + SP balance) and `ValidateMetamagicOptions` (per-option spell shape prerequisites — Twinned naturally rejects). Errors return early so no slot is burnt.
   - Compute total cost.
2. After pending-save creation (before the existing pending-save persistence loop), apply:
   - **Careful**: build a set from `CarefulTargetIDs`, capped at `CarefulSpellCreatureCount(cha)`. For each match, set `PendingSave.AutoSuccess=true`. The save row is still persisted to satisfy `ResolveAoEPendingSaves`' "all rolled" check; status will be marked as a success.
   - **Heightened**: set `PendingSave.Disadvantage=true` on the **first** affected creature (canonical order: the order the affected list comes back from `FindAffectedCombatants`). The /save handler will pick this up via the in-memory `AoECastResult.PendingSaves` slice; DB persistence of the flag is out of scope (no advantage column exists; spec line 946 just needs the flag to flow through `AoECastResult` for the surface).
   - **Empowered**: set `result.IsEmpowered=true` and `result.EmpoweredRerolls = CHA mod`. The damage application stage (`ResolveAoESaves` / `ApplyDamage`) is the consumer; for this finding we surface the flag on `AoECastResult`. (Reroll execution lives in SR-025 prompt territory — but at minimum we wire the field so it's no longer ignored.)
3. After slot deduction, deduct SP via `DeductFeaturePool` (same call pattern as `Cast`).
4. Populate `AoECastResult` with: `CarefulSpellCreatures` (from CHA), `IsEmpowered`, `EmpoweredRerolls`, `IsHeightened`, `IsSubtle`, `ExtendedDuration`, `DistantRange`, `MetamagicCost`, `SorceryPointsRemaining`. Add these fields to `AoECastResult` (mirror naming of `CastResult`).
5. Add new fields on `PendingSave` struct: `AutoSuccess bool`, `Disadvantage bool`. They're informational on the in-memory return; no DB column change.
6. **Twinned**: validated rejection already comes out of `validateTwinnedSpell` (AoE → error). Add an integration test asserting the rejection error message contains "area of effect".

## Reviewer notes
- **Cylinder switch** (`aoe.go:250-251`): correctly delegates to `SphereAffectedTiles(targetCol, targetRow, aoe.RadiusFt)`. For a 2D combat grid, a cylinder's tile footprint IS the disc of the same radius — `height_ft` only matters for vertical clearance / LoS, which we don't model. Confirmed by `TestGetAffectedTiles_Cylinder` (cylinder ≡ sphere disc for r=20) and `TestCastAoE_Cylinder_Moonbeam` (Moonbeam at r=5 selects target tile + cardinals).
- **`height_ft` silently dropped**: `AreaOfEffect` struct (`aoe.go:21-27`) intentionally omits `HeightFt`. Seeded JSON `{"shape":"cylinder","radius_ft":5,"height_ft":40}` parses fine (extra field ignored). Acceptable now — no code path currently reasons about vertical clearance. Follow-up belongs to SR-008 (FoW vertical) / SR-014 (zone overlays). Worth a one-line `// TODO(SR-008/SR-014): add HeightFt when 3D clearance lands` later, but not blocking.
- **`AoECastCommand.Metamagic` / `CarefulTargetIDs`** (`aoe.go:344-348`): mirrors `CastCommand` shape.
- **Twinned rejection** (`metamagic.go:108-115`): `validateTwinnedSpell` rejects when `hasAreaOfEffect(spell)` returns true (any `spell.AreaOfEffect.Valid` with non-empty RawMessage). Pre-slot ordering at `aoe.go:418-440` confirmed — `ValidateMetamagicOptions` runs before slot deduction. `TestCastAoE_Twinned_Rejected` asserts the error contains "area of effect" and the slot is NOT deducted.
- **Heightened RAW** (`aoe.go:520-531`): assigns `Disadvantage=true` to the first affected combatant who is not also a Careful auto-success. Matches PHB Heightened ("give one target … disadvantage on its first saving throw"). The first-target rule is canonical by `FindAffectedCombatants` iteration order (which is the order returned by `ListCombatantsByEncounterID`). Note: this is deterministic per DB row order, not per AoE geometry — acceptable for now since no SR-025 prompt exists yet to pick the target. `TestCastAoE_Heightened_FirstTargetDisadvantage` covers it.
- **Careful cap enforcement** (`aoe.go:505-515`): hard-cap via `if i >= carefulCap { break }`. Truncation is by first-N input order, NOT silent — extra IDs simply do not enter `carefulSet`. `TestCastAoE_Careful_SparesAlly` asserts `result.CarefulSpellCreatures == 4` (CHA 18 → mod 4). A negative test for the cap (e.g. 5 IDs with cap 4) would strengthen coverage but is not blocking; existing assertion proves the cap value flows through.
- **Careful DB auto-resolve** (`aoe.go:558-566`): the Careful row is created with `Status="pending"` then immediately updated via `UpdatePendingSaveResult` (RollResult=DC, Success=true). This keeps `ResolveAoEPendingSaves`' "all rolled" gate working without waiting on a player roll. `TestCastAoE_Careful_SparesAlly` asserts the ally's row ID appears in resolved-successes list.
- **Empowered**: flag-only surface (`aoe.go:683-684`). `ResolveAoESaves` (aoe.go:874-931) rolls damage once and never consults `result.IsEmpowered` / `EmpoweredRerolls`. **Re-roll is NOT executed.** This matches the worker plan's explicit out-of-scope note ("surfaced flag only … SR-025 prompt territory"). Single-target `Cast` has the same gap — `applyMetamagicEffects` also only surfaces the flag. Acceptable scope-wise but worth tracking in SR-025.
- **DeductFeaturePool / FeatureUse shape** (`aoe.go:425-440`, `aoe.go:601-607`): correctly uses `ParseFeatureUses(char, FeatureKeySorceryPoints)` returning `map[string]character.FeatureUse` (SR-009 unified shape — Current/Max/Recharge struct, not flat int). `DeductFeaturePool` (`feature_integration.go:53-59`) preserves Max+Recharge metadata via `SetFeaturePool`. Confirmed.
- **Pre-slot vs post-slot ordering**: metamagic validation at step 6b (pre-slot), SP deduction at step 16a (post-slot). On a rejected metamagic the slot is not burnt (Twinned test asserts `slotDeducted == false`). On a successful metamagic but rejected slot deduction, SP would also not be burnt because slot deduction occurs first. Correct ordering.
- **Early-return style** (CLAUDE.md): all error paths use early-return. The metamagic block at `aoe.go:425-440` is a flat sequence of `if err != nil { return ... }`, no nested conditionals. Compliant.
- **No over-engineering**: cylinder reuses `SphereAffectedTiles` (no new helper), `applyAoEMetamagicEffects` is a near-twin of `applyMetamagicEffects` — kept separate only because `AoECastResult`/`CastResult` are distinct types (acknowledged in comment at aoe.go:672-674). Acceptable duplication; a future refactor could lift via interface but adds churn now.
