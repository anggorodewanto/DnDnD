## Finding (verbatim from review)
Escape contested check is STR-only; spec allows STR or DEX.

## Acceptance criteria
- The escape contested check accepts STR (Athletics) or DEX (Acrobatics) at the escaping creature's choice, per spec.
- Test: escape using Acrobatics succeeds when DEX is the higher mod; escape using Athletics still works.
- Tests pass; `make cover-check` green.

## Worker plan
- Add `UseAthletics bool` field to `EscapeCommand` so we can distinguish explicit-athletics from no-choice.
- In `Escape()`, when neither `UseAcrobatics` nor `UseAthletics` is set, fetch both STR and DEX mods and pick the higher one (ties go to STR per convention).
- Update `dispatchEscape` handler to also detect `--athletics` in args.
- Write test `TestEscape_AutoPickHigherMod` verifying DEX is auto-selected when DEX mod > STR mod.
- Verify existing tests still pass (explicit `UseAcrobatics: true` and `UseAcrobatics: false` with `UseAthletics: false` should still work since explicit athletics = old default behavior).
- Run `go test ./...` and `make cover-check`.
