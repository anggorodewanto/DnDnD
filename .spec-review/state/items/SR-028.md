## Finding (verbatim from review)
OA: no DM dashboard prompt for DM-controlled hostiles; no end-of-round forfeiture sweep. Move handler posts only to `#your-turn` (PC path); DM-controlled hostile OAs silently dropped.

## Acceptance criteria
- When a PC's move provokes an OA from a DM-controlled hostile, a DM dashboard prompt is posted (not `#your-turn`).
- An end-of-round sweep forfeits any OA that the DM (or player) didn't act on.
- Test: PC moves out of hostile reach → DM-side prompt appears; if unanswered by end of round, the reaction is forfeited and the prompt cleared.
- Tests pass; `make cover-check` green.

## Worker plan

1. **dmqueue**: add `KindOpportunityAttack` EventKind + kindLabel entry (⚔️ Reaction) so OA prompts render distinctly in `#dm-queue`.
2. **combat.Service**: add an in-memory tracker for pending OA item IDs keyed by encounter ID (`pendingOAsMu`/`pendingOAsByEncounter`), with:
   - `RecordPendingOA(encounterID, itemID)` (called by move handler after a successful DM-queue Post).
   - `ForfeitPendingOAs(ctx, encounterID)` — drains the slice and calls `dmNotifier.Cancel(ctx, itemID, "forfeited at end of round")` for each. Nil notifier → silent drain. Errors swallowed (best-effort).
   - Fire `ForfeitPendingOAs` from `advanceRound` (both natural and skip-all branches already go through `advanceRound`).
3. **discord.MoveHandler**: 
   - Add `MoveOANotifier` interface (`Post`/`Cancel` subset of `dmqueue.Notifier`) and `MoveOAReactionRecorder` interface (`RecordPendingOA`).
   - Add `SetOpportunityAttackNotifier(...)` setter that wires both.
   - In `fireOpportunityAttacks`, branch on `hostile.IsNpc`:
     - PC ally / mover-NPC path (PC hostile): existing `#your-turn` post (regression-preserved).
     - DM-controlled hostile (`hostile.IsNpc == true`): build a `dmqueue.Event{Kind: KindOpportunityAttack, ...}` and Post via notifier; record returned itemID via recorder. Skip the `#your-turn` post for that hostile.
   - Resolve guild+campaign via the existing `campaignProv` (`GetCampaignByGuildID`) using the encounter→guild path. Best-effort: missing notifier/recorder → fall back silently to no-op for DM-controlled hostiles (no #your-turn duplicate).
4. **Wiring** (`cmd/dndnd/discord_handlers.go`): call `handlers.move.SetOpportunityAttackNotifier(deps.notifier, deps.combatService)` when both are non-nil (after `SetOpportunityAttackHooks`).
5. **Tests** (red/green TDD):
   - `move_handler_test.go`: PC moves out of DM-controlled hostile reach → notifier captures `KindOpportunityAttack` Post; `#your-turn` channel send count is 0 for that trigger.
   - `move_handler_test.go`: PC moves out of PC ally reach → `#your-turn` receives one post (regression guard).
   - `combat/opportunity_attack_test.go` (new) or `initiative_test.go`: `RecordPendingOA` + `ForfeitPendingOAs` round-trips; advanceRound calls notifier.Cancel for each pending itemID.
6. Run `go test ./...` and `make cover-check`.
